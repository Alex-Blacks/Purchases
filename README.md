# Purchases
Жена попросила создать программу для покупок, посмотрим что из этого выйдет


Описание: Программа должна хранить список покупок, чтобы можно было заполнять на одном устройстве, и читать на другом. Также там должна быть история, и какая-никая статистика.

1) Для начала нужно сделать базу:
users:
id: id пользователя,
name: имя пользователя,
password: пароль,
email: почта пользователя, для восстановления пароля,
role: пользователь или админ,
status: активный или в блоке,
created_at: дата создания,
update_at: дата обновления

categories:
id: id категории,
name: название категории,

products:
id: id товара,
title: название товара,
unit: мера измерения товара
category_id: категория товара,

product_aliases:
id: id алиаса,
product_id: id продукта,
alias: псевдоним продукта,

stores:
id: id магазина,
name: название магазина,

orders:
id: id заказа,
users_id: пользователь который заказал,
store_id: из какого мазазина,
created_at: дата создания,

order_items:
id: id позиции,
order_id: какой заказ,
products_id: какой товар,
count: количество,

purchases:
id: id покупки,
user_id: пользователь который заказал покупку,
external_id: внешний id,
store_id: магазин в котором купили,
total_sum: сумма покупки,
purchased_at: дата покупки,
raw_qr: qr код,

purchase_items:
id: id позиции,
purchase_id: какая покупка
row_name: товар который купили,
product_id: добавляем, чтобы можно было нормализовывать имена потом,
count: количество купленного,
price: цена по которой купили,

2) Дальше начинаю backend на Go писать.
    2.0) Логика такая: Сервис должен добовлять\удалять\изменять позиции в каждой таблице кроме истории, а также получать данные из всех таблиц.
